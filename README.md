# HO12 — Hosted Dashboard UI

> Hands-on portfolio project · **Week 5** · **Solo** · module M16. Part of the **ForgeMind AI — AI Productivity Essentials** course.

## Goal

**Done when:** A hosted dashboard (cards + a chart) on real data

## What to ship

The HTML/JS/CSS source + README with the live URL + a screenshot.

## Pick a problem statement

Choose **one** of these real use-cases — or bring your own (get it approved first):

1. You track habits, workouts and spending in scattered notes and never see the big picture. Build a hosted dashboard that reads your real data and shows summary cards (streaks, totals) plus a trend chart over time. Success: a live URL with the cards and chart populated from your data, and a screenshot in the README.

2. Your team's KPIs live in a shared sheet nobody opens. Build a hosted dashboard that pulls that sheet or an API and shows the key numbers as cards with a chart for the main metric. Success: a live URL where the cards and chart reflect the real sheet data, with the data source and screenshot in the README.

3. As a creator you cannot tell which content is actually working. Build a hosted dashboard fed by your content data that shows views and engagement as cards plus a chart of performance over time. Success: a live URL displaying your real numbers with a trend chart, and the data source plus a screenshot in the README.

4. You are curious about a public dataset (your city's open data, sports stats, or app reviews) but the raw data is unreadable. Build a hosted dashboard that pulls it and shows the headline figures as cards with a chart. Success: a live URL where cards and chart render the real public data, documented with a screenshot in the README.

5. Your projects' status is buried in a task tool and stakeholders keep asking where things stand. Build a hosted dashboard fed by your task data that shows tasks-by-status cards and a burndown chart. Success: a live URL where the cards and chart reflect the real project data, with the data source and a screenshot in the README.
